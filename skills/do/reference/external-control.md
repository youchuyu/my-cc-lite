# External Control

本文件定义 `/do` 的外部高阶接管流程。所有状态写入必须遵守 `state-boundary.md`。

选择外部高阶接管时，`/do` 不进入 my-cc-lite 原生 task loop，而是将完整执行上下文交给外部流程编排。

外部流程拥有 task 执行结果的判定权和执行状态推进权，但不拥有 task 结构修改权。它只能更新 `tasks[].status`、`tasks[].statusReason`，以及由脚本派生的顶层 `status` 和 `updatedAt`。

## 阶段总览

流程阶段如下：

1. 进入与交接上下文
2. 外部流程编排执行
3. 写入执行状态
4. 中断、恢复与停止

## 1. 进入与交接上下文

### 工作内容

判断本轮 `/do` 是否进入外部高阶接管，并向外部流程提供完整执行上下文。

提供给外部流程的上下文包括：

- 当前 `plan.md` 摘要。
- 完整 `task.json.tasks[]`。
- 当前顶层 `status`、`stage` 和 `verification` 摘要。
- 执行边界和 `state-boundary.md` 中的状态写入规则。

### 进入条件

- 首次物化成功后，用户选择外部高阶接管。

### 边界

外部高阶接管只发生在首次物化成功后的接管方式选择中。已有 `task.json` 的恢复执行不重新进入外部高阶接管，而是回到 my-cc-lite 原生接管。

### 跳转

进入“外部流程编排执行”。

## 2. 外部流程编排执行

### 工作内容

外部流程接管完整 `tasks[]`，自行决定如何推进多个 task，例如：

- 按顺序执行。
- 并行分析后串行写入结果。
- 跨 task 共享上下文。
- 委派自己的 agent/helper。
- 对多个 task 持续推进状态。
- 一次性执行完整 `tasks[]`。

### 边界

外部流程只拥有执行编排权、结果判定权和执行状态推进权，不修改任务结构、计划内容或后续阶段状态。

### 跳转

- 如果外部流程形成了可映射到现有 task id 的执行结果，进入“写入执行状态”。
- 如果外部流程无法可靠映射到现有 `tasks[]`，进入“中断、恢复与停止”。
- 如果外部流程需要改变任务结构、计划内容或后续阶段状态才能继续，进入“中断、恢复与停止”。
- 如果需要用户确认业务取舍、权限、外部账号、破坏性操作或计划范围调整，进入“中断、恢复与停止”。

## 3. 写入执行状态

### 工作内容

外部流程可以通过受限接口持续落盘执行状态：

```text
scripts/run.mjs do update-task
```

### 规则

写入规则以 `state-boundary.md` 为准。外部流程不得绕过脚本直接修改 `task.json`。

### 跳转

写入后，外部流程可以继续编排执行；如果已无可推进 task，或继续执行需要用户确认，则停止并说明结果。

## 4. 中断、恢复与停止

### 必须停止的情况

外部流程可以自行决定如何连续推进完整 `tasks[]`，但遇到以下情况必须停止并说明原因：

- 外部流程无法可靠映射到现有 `tasks[]`。
- 外部流程需要新增、删除、重排、合并或拆分 task 才能继续。
- 外部流程需要修改 `plan.md`、`verification`、`archive` 或任务结构。
- 需要用户确认业务取舍、权限、外部账号、破坏性操作或计划范围调整。

### 恢复规则

停止前已经形成的执行状态，仍只能通过受限 `update-task` 接口落盘到现有 task。

如果外部流程中断，后续 `/do` 看到已有 `task.json` 后，只由 my-cc-lite 原生恢复接管，根据已经落盘的 task 状态继续推进。
