# 外部高阶流程接管

本文件定义 `/do` 的外部高阶接管流程。所有状态写入必须遵守 `state-boundary.md`。

选择外部高阶接管时，`/do` 不进入 my-cc-lite 原生 task loop，而是将完整执行上下文交给外部流程编排。

外部流程拥有 task 执行结果的判定权和执行状态推进权，但不拥有 task 结构修改权。它只能更新 `subtasks[].status`、`subtasks[].statusReason`，以及由脚本派生的顶层 `status` 和 `updatedAt`。

## 阶段总览

流程阶段如下：

1. 进入与交接上下文
2. 外部流程编排执行
3. 写入执行状态
4. 中断、恢复与停止

## 1. 进入与交接上下文

### 工作内容

判断本轮 `/do` 是否进入外部高阶接管，并向外部流程提供完整执行上下文。

提供给外部流程的上下文包括：

- 当前 `plan.md` 摘要。
- 完整 `task.json.subtasks[]`。
- 当前顶层 `status`、`stage` 和 `verification` 摘要。
- 执行边界和 `state-boundary.md` 中的状态写入规则。

### 进入条件

- 首次物化成功后，用户选择外部高阶接管。

### 边界

外部高阶接管只发生在首次物化成功后的接管方式选择中。已有 `task.json` 的后续执行不重新进入外部高阶接管；`/do` 先恢复状态并选择当前 task，只有用户明确继续执行时才回到 my-cc-lite 原生接管。

### 跳转

进入“外部流程编排执行”。

## 2. 外部流程编排执行

### 工作内容

外部流程接管完整 `subtasks[]`，自行决定编排策略，例如顺序执行、并行分析后串行写入，或一次性推进完整 `subtasks[]`。

执行状态通过 `scripts/run.mjs do update-task` 落盘，不得绕过脚本直接修改 `task.json`。

### 边界

外部流程只拥有执行编排权、结果判定权和执行状态推进权，不修改任务结构、计划内容或后续阶段状态。

### 跳转

- 如果外部流程形成了可映射到现有 task id 的执行结果，继续编排或停止并说明结果。
- 如果外部流程无法可靠映射到现有 `subtasks[]`，进入”中断、恢复与停止”。
- 如果外部流程需要改变任务结构、计划内容或后续阶段状态才能继续，进入”中断、恢复与停止”。
- 如果需要用户确认业务取舍、权限、外部账号、破坏性操作或计划范围调整，进入”中断、恢复与停止”。

## 3. 中断、恢复与停止

### 必须停止的情况

外部流程可以自行决定如何连续推进完整 `subtasks[]`，但遇到以下情况必须停止并说明原因：

- 外部流程无法可靠映射到现有 `subtasks[]`。
- 外部流程需要做 state-boundary.md 禁止的结构修改才能继续。
- 需要用户确认业务取舍、权限、外部账号、破坏性操作或计划范围调整。

### 恢复规则

停止前已经形成的执行状态，仍只能通过受限 `update-task` 接口落盘到现有 task。

如果外部流程中断，后续 `/do` 看到已有 `task.json` 后，先进入恢复状态检查；只有用户明确继续执行时，才由 my-cc-lite 原生接管根据已经落盘的 task 状态继续推进。
